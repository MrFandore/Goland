# Практика 2 - Структура Go проекта

## Структура проекта
```
Practica_2/
├── cmd/
│   └── myapp/
│       └── main.go
├── internal/
│   └── app/
│       └── app.go
├── utils/
│   └── logger.go
├── go.mod
├── .gitignore
└── README.md
```

## Описание файлов
- **cmd/myapp/main.go** - главный файл для запуска приложения
- **internal/app/app.go** - основная логика сервера и маршруты
- **utils/logger.go** - функции для логирования запросов
- **go.mod** - файл зависимостей Go

## Запуск проекта

### 1. Скачайте проект
```bash
git clone https://github.com/MrFandore/Goland.git
cd Goland/Practica_2
```

### 2. Установите зависимости
```bash
go mod download
```

### 3. Запустите сервер
```bash
go run cmd/myapp/main.go
```

## Проверка работы

После запуска сервер доступен по адресу: http://localhost:8080

### Проверка эндпоинтов:

**Главная страница:**
```
http://localhost:8080/
```
Ожидаемый результат: текст "Hello, Go project structure!"

**Статус сервера:**
```
http://localhost:8080/ping
```
Ожидаемый результат: JSON со статусом "ok" и текущим временем

### Проверка через командную строку:
```bash
curl http://localhost:8080/
curl http://localhost:8080/ping
```

## Особенности проекта
- Логирование всех входящих запросов в консоль
- Правильная структура Go проекта с разделением на пакеты
- Использование internal директории для приватного кода
- Отдельная утилита для логирования в пакете utils

## Требования
- Установленный Go версии 1.25 или выше
- Установленный Git

## Решение проблем

**Порт занят:**
Измените порт в файле internal/app/app.go с ":8080" на ":8081"

**Go не установлен:**
Скачайте и установите Go с https://golang.org/dl/

**Файлы не найдены:**
Убедитесь, что вы в папке Practica_2 и присутствует файл go.mod

## Остановка сервера
Нажмите Ctrl+C в командной строке где запущен сервер
