# Репозиторий практических работ по Go

Репозиторий содержит практические работы по дисциплине "Технологии создания программного обеспечения" для изучения языка программирования Go.

## Содержание репозитория

### Практическое занятие №1
**Тема:** Установка и настройка окружения Go

**Содержание:**
- Установка Go и Git
- Инициализация Go-модуля
- Создание минимального HTTP-сервера
- Подключение внешних зависимостей (UUID)
- Сборка и запуск приложения

**Ключевые технологии:**
- `net/http` - стандартная библиотека HTTP
- `github.com/google/uuid` - генерация UUID
- Go modules - управление зависимостями

### Практическое занятие №2  
**Тема:** Структура Go-проекта

**Содержание:**
- Организация проекта по стандартам сообщества
- Создание структуры каталогов (cmd/, internal/, pkg/)
- Реализация модульного HTTP-сервера
- Middleware для логирования и обработки запросов
- Единые форматы JSON-ответов

**Ключевые технологии:**
- Project layout standards
- HTTP middleware
- JSON сериализация
- Модульная архитектура

### Практическое занятие №3
**Тема:** Реализация HTTP-сервера на стандартной библиотеке

**Содержание:**
- Работа с net/http без фреймворков
- Обработка GET/POST запросов
- Чтение query-параметров и тела запроса
- In-memory хранилище данных
- Собственная реализация middleware

**Ключевые технологии:**
- http.ServeMux
- JSON decoding/encoding
- In-memory storage с sync.RWMutex
- Custom middleware chain

### Практическое занятие №4
**Тема:** Маршрутизация с chi и создание CRUD-сервиса

**Содержание:**
- Маршрутизация HTTP-запросов с роутером chi
- Построение REST API с методами GET/POST/PUT/DELETE
- Реализация CRUD-сервиса "Список задач"
- Middleware для логирования и CORS
- Тестирование API через curl/Postman

**Ключевые технологии:**
- Chi router
- REST API design
- CRUD operations
- Middleware (logging, CORS)
- API testing

## Технические требования

- **Go:** версия 1.21 или выше
- **Git:** для управления версиями
- **IDE:** GoLand, VS Code с Go extension или аналоги
- **Тестирование:** curl, Postman, HTTPie

## Структура проектов

Проекты организованы согласно стандартам Go-сообщества:

project/
├── cmd/              # Входные точки приложений
├── internal/         # Приватный код проекта
│   ├── app/         # Основная логика приложения
│   ├── api/         # HTTP обработчики
│   └── storage/     # Логика хранения данных
├── pkg/             # Публичные библиотеки
│   └── middleware/  # Middleware компоненты
├── go.mod          # Модуль Go
└── README.md       # Документация


## Навыки и компетенции

По завершению практик студент будет уметь:

- Устанавливать и настраивать окружение для разработки на Go
- Создавать структурированные Go-проекты согласно стандартам
- Реализовывать HTTP-серверы на стандартной библиотеке
- Использовать сторонние роутеры (chi) для сложной маршрутизации
- Строить REST API с полным набором CRUD-операций
- Реализовывать middleware для сквозной функциональности
- Тестировать API с помощью инструментов командной строки
- Работать с системой модулей Go и внешними зависимостями

## Запуск проектов

Каждый проект содержит собственные инструкции по запуску в соответствующих директориях. Основные команды:

```bash
# Инициализация модуля
go mod init example.com/project-name

# Запуск в режиме разработки
go run ./cmd/server

# Сборка бинарного файла
go build -o bin/server ./cmd/server

# Тестирование API
curl http://localhost:8080/endpoint
```
