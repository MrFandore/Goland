# Репозиторий Пряшникова Дмитрия Максимовича

## Описание репозитория

Данный репозиторий содержит практические работы по дисциплине **"Технологии создания программного обеспечения"** за 1 семестр 2025-2026 учебного года. Все проекты реализованы на языке программирования Go и демонстрируют различные аспекты разработки backend-приложений.

## Структура проектов

### Практика 1 - Базовый HTTP сервер
**Цель:** Изучение основ создания HTTP-сервера на Go
- Установка и настройка окружения Go
- Создание минимального HTTP-сервиса
- Работа с внешними зависимостями
- Базовые маршруты и JSON-ответы

### Практика 2 - Структура Go проекта
**Цель:** Освоение правильной организации Go проекта
- Разделение кода по пакетам (cmd/, internal/, pkg/)
- Создание модульной архитектуры
- Логирование запросов
- Правильное импортирование пакетов

### Практика 3 - CRUD API с памятью
**Цель:** Реализация полноценного CRUD API
- Создание, чтение, обновление и удаление задач
- In-memory хранилище данных
- Middleware для логирования
- Валидация входных данных

### Практика 4 - Продвинутая маршрутизация с Chi
**Цель:** Изучение сторонних роутеров и middleware
- Использование Chi роутера
- Группировка маршрутов
- CORS middleware
- Полнофункциональное REST API

### Практика 5 - Работа с PostgreSQL
**Цель:** Изучение взаимодействия с реляционными базами данных
- Подключение к PostgreSQL с использованием database/sql
- Выполнение параметризованных запросов (INSERT, SELECT)
- Использование транзакций для массовых операций
- Настройка пула соединений и работа с контекстом
- Реализация репозитория для работы с задачами

### Практика 6 - Использование ORM GORM для работы с PostgreSQL
**Цель:** Освоение работы с ORM для автоматизации операций с базой данных
- Настройка GORM для работы с PostgreSQL
- Автоматические миграции и создание таблиц
- Определение связей между моделями (один-ко-многим, многие-ко-многим)
- Использование Preload для загрузки связанных данных
- Валидация данных и обработка ошибок
- Настройка пула соединений с базой данных

### Практика 7 - Работа с Redis для кэширования данных
**Цель:** Изучение использования in-memory хранилища для кэширования
- Установка и настройка Redis сервера
- Выполнение базовых операций (SET, GET, TTL)
- Реализация кэширования данных с автоматическим истечением
- Интеграция Redis с Go приложениями
- Использование Redis для хранения сессий и кэширования результатов БД

### Практика 8 - Работа с MongoDB для документного хранения данных
**Цель:** Освоение работы с документо-ориентированной СУБД
- Запуск MongoDB в Docker с использованием docker-compose
- Проектирование документной модели данных
- Реализация CRUD операций с использованием официального драйвера MongoDB
- Создание индексов и выполнение поисковых запросов
- Обработка ошибок и настройка таймаутов
- Работа с BSON и ObjectID

### Практика 9 - Регистрация и аутентификация пользователей с bcrypt
**Цель:** Реализация системы безопасной аутентификации
- Хэширование паролей с использованием алгоритма bcrypt
- Валидация входных данных (email, пароль)
- Регистрация новых пользователей с проверкой уникальности email
- Аутентификация пользователей путем сравнения хэшей паролей
- Настройка стоимости хэширования для баланса безопасности и производительности
- Обработка ошибок и возврат соответствующих HTTP статусов

### Практика 10 - JWT аутентификация и авторизация
**Цель:** Реализация современной системы аутентификации с использованием токенов
- Генерация access и refresh токенов
- Реализация middleware для аутентификации и авторизации
- Настройка RBAC (ролевой модели доступа) и ABAC (атрибутивной модели)
- Обновление токенов с использованием refresh токенов
- Реализация rate limiting для защиты от brute-force атак
- Создание blacklist для refresh токенов
- Работа с RS256 алгоритмом подписи JWT

## Технические особенности

Все проекты демонстрируют:
- **Чистую архитектуру** - разделение ответственности между компонентами
- **Правильную структуру** - организация кода по стандартам Go сообщества
- **Обработку ошибок** - корректные HTTP статусы и JSON-ответы
- **Логирование** - отслеживание запросов и ошибок
- **Валидацию** - проверку входных данных
- **Работу с базами данных** - использование PostgreSQL, MongoDB, Redis
- **Безопасность** - хэширование паролей, JWT аутентификация, защита от инъекций
- **Производительность** - кэширование с Redis, оптимизация запросов, пулы соединений

## Требования для запуска

Для работы с проектами необходимо:
1. Установленный Go (версии 1.21 или выше)
2. Установленный Git
3. Любой текстовый редактор или IDE
4. Для Практики 5-6: установленный PostgreSQL
5. Для Практики 7: установленный Redis сервер
6. Для Практики 8: установленные Docker и Docker Compose
7. Для Практики 9-10: понимание принципов аутентификации и авторизации

## Как использовать

Каждая практическая работа находится в отдельной папке (Practica_1, Practica_2 и т.д.) и содержит:
- Исходный код проекта
- Инструкции по запуску
- Примеры тестирования API
- Документацию по использованию

### Дополнительные требования по практикам:

**Для Практики 5-6:**
- Создать базу данных PostgreSQL
- Настроить соединение через переменные окружения
- Выполнить SQL-скрипт для создания таблиц (для Практики 5)

**Для Практики 7:**
- Установить и запустить Redis сервер
- Настроить подключение к Redis

**Для Практики 8:**
- Запустить MongoDB через Docker Compose
- Настроить переменные окружения для подключения

**Для Практики 9-10:**
- Создать базу данных для пользователей
- Настроить секретные ключи для JWT
- Следовать рекомендациям по безопасности (HTTPS, rate limiting)

## Технологический стек

- **Язык программирования:** Go 1.21+
- **Фреймворки и библиотеки:** Chi, GORM, JWT, bcrypt
- **Базы данных:** PostgreSQL, MongoDB, Redis
- **Контейнеризация:** Docker, Docker Compose
- **Инструменты разработки:** Git, curl/Postman для тестирования
